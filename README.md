# English & TOEFL Speaking Coach

A fully local web app for practicing English conversation and TOEFL iBT Speaking tasks. Speech-to-text runs
locally on CPU via faster-whisper; feedback is generated by your local FreeLLMAPI instance (OpenAI-compatible
endpoint); feedback and model answers are read aloud with the browser's built-in `speechSynthesis` API.

## Architecture

- **backend/** — FastAPI server (Python). Accepts recorded audio, transcribes it with faster-whisper, sends the
  transcript + task context to FreeLLMAPI, stores every attempt in a local SQLite file, and serves history/stats.
- **frontend/** — React + Vite app. Records audio with `MediaRecorder`, drives prep/speaking timers, talks only to
  the local backend, and plays feedback aloud with `speechSynthesis`.

Nothing here talks to the public internet except `npm install` / `pip install` during setup. Once running, all
requests stay on `localhost`.

## 1. Prerequisites

- Python 3.10+ 
- Node.js 18+
- FreeLLMAPI already running locally at `http://localhost:3001` (started separately via `docker compose up -d`
  in your freellmapi folder) with a unified key created on its Keys page.

## 2. Backend setup

```bash
cd backend
python -m venv .venv

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
# macOS/Linux
source .venv/bin/activate

pip install -r requirements.txt
```

This installs `faster-whisper` for you. If you ever need to install it manually:

```bash
pip install faster-whisper
```

Start the backend:

```bash
uvicorn main:app --reload --port 8000
```

The first time you submit a recording, faster-whisper will download the selected model (tiny/base/small) to a
local cache — this requires internet once, then works fully offline.

## 3. Frontend setup

In a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open the URL Vite prints (default `http://localhost:5173`).

## 4. Where to paste your FreeLLMAPI unified key

Open the app → **Settings** page (`http://localhost:5173/settings`) → paste your key (format
`freellmapi-xxxxxxxx`) into the **"FreeLLMAPI unified key"** field → click **Save settings**. It's written to
`backend/data/config.json` on your machine only, and is never printed to any log.

On the same page you can also change:
- **Base URL** — default `http://localhost:3001/v1`, in case you ever move FreeLLMAPI.
- **Model** — default `auto` (router picks the best free model); type an exact model name to pin one.
- **faster-whisper model size** — `tiny` / `base` / `small` (default `small`; switch to `base` or `tiny` if
  transcription feels slow on your CPU).

## 5. What's running where

| Component   | URL                        | Notes                                            |
|-------------|-----------------------------|---------------------------------------------------|
| Frontend    | http://localhost:5173       | React + Vite dev server                           |
| Backend     | http://localhost:8000       | FastAPI, talks to Whisper + FreeLLMAPI            |
| FreeLLMAPI  | http://localhost:3001       | Started separately by you (`docker compose up -d`)|
| SQLite DB   | `backend/data/app.db`       | Session history, created automatically            |
| Settings    | `backend/data/config.json`  | Your key/URL/model choices, local file only       |

## 6. Error handling you'll see in the app

- **FreeLLMAPI unreachable** — "Could not reach FreeLLMAPI... run 'docker compose up -d' in your freellmapi
  folder." Shown on Home, Settings, and after a failed practice attempt.
- **Missing/invalid key** — "Your FreeLLMAPI unified key is missing or invalid... get one from the Keys page at
  http://localhost:3001."
- **Mic permission denied** — "Microphone permission was denied. Please allow microphone access for this site in
  your browser settings and try again."
- **faster-whisper not installed** — Home/Settings will show "faster-whisper is not installed. Run: pip install
  faster-whisper."

## 7. Practice modes

**General English Practice** — 24 everyday topics (daily life, opinions, past experiences, future plans,
describing people/places). Record an answer of any length; get plain-language feedback on grammar, vocabulary,
and clarity, plus an improved rewritten version read aloud.

**TOEFL Speaking Practice** — matches the redesigned TOEFL iBT Speaking section that launched
January 21, 2026 (two tasks, ~8 minutes total, no prep time):
- **Listen and Repeat**: 7 short sentences per set (tied to a picture), heard once via speechSynthesis, repeated
  back immediately with no prep — 8–12 seconds of recording per sentence, sentences get progressively longer.
- **Take an Interview**: 4 questions per set on one familiar topic, heard via speechSynthesis, answered immediately
  with no prep — 45 seconds of recording per question.

Each attempt returns a strict-JSON rubric response scored 1–6: a score band with a one-line reason, Fluency /
Accuracy / Pronunciation-Delivery (Listen and Repeat) or Fluency / Accuracy / Coherence (Take an Interview),
a sentence-by-sentence or question-by-question breakdown, and one focus point for next time (auto-read aloud in
an en-US voice).

## 8. History

The **History** page lists every past session (both modes), lets you reopen any of them, and shows a score-trend
chart for TOEFL attempts over time.
